Overview

This project focused on evaluating the cTAKES clinical NLP pipeline for identifying COVID-19 related symptoms from de-identified Electronic Health Record notes. The work brought together cloud-stored clinical documents, FHIR-based DocumentReference metadata, structured cTAKES outputs, human-annotated reference data, SQL integration, Python-based evaluation, and Power BI visualization to create an end-to-end framework for assessing symptom extraction quality.

Implementation

The pipeline began with the preparation and cleaning of cTAKES output files, which contained symptom extractions stored as JSON objects. These files were standardized to ensure consistent formatting across all entries. FHIR DocumentReference files were then parsed to map document identifiers to their source notes stored securely in cloud storage.

A relational SQL master table was constructed to consolidate all relevant components. Each row linked a symptom extraction to its encounter, subject, document identifier, and original JSON payload. JSON fields were unpacked into structured SQL columns that captured symptom text, polarity, and coded clinical concepts. This architecture enabled efficient querying and filtering of extraction attributes.

Human annotation files were incorporated into the workflow to serve as reference labels for validation. Using Python, batches of source notes were retrieved from S3, aligned with the annotation data, and matched to the corresponding cTAKES extractions through the SQL mapping table. A batch evaluation process classified each extraction as correct, incorrect, or missed, allowing the workflow to scale without requiring redesign.

After evaluation, the structured SQL tables were connected to Power BI to support interactive visualization. This environment enabled detailed exploration of symptom trends, extraction patterns, and data quality signals across the dataset.

Results

The completed pipeline produced an integrated and analysis-ready dataset that unified NLP outputs, document metadata, and annotation references. The evaluation stage clarified the behavior of cTAKES by distinguishing correct detections, missed findings, and inconsistent predictions. Power BI dashboards provided visual insights into frequently identified symptoms, encounter-level trends, and overall data consistency, enabling more efficient review and assessment of clinical NLP performance.
