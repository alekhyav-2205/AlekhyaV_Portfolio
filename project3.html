DiaHealth: Diabetes Care Relational Database

Database Architecture

Role

Database Architect

Tech Stack

SQL, PostgreSQL, FHIR Modeling

Domain

Chronic Disease Management

Overview

DiaHealth is a clinical data architecture designed to support a diabetes health-coaching program by integrating EHR clinical indicators with coaching-application data. The project structures patient enrollment, clinical observations, lifestyle interventions, and six-month outcome assessments.

The schema follows FHIR-aligned patterns, modeling entities such as Patient, Observation, Encounter, Practitioner, and CarePlan. Data is organized into a longitudinal structure for analyzing changes in HbA1c, lipid profile, BMI, and blood pressure across intervention stages.

The design incorporates HIPAA-aligned principles, separating PHI from analytical identifiers and defining clear access assumptions for clinical and analytical roles.

Schema Design & Architecture

FHIR-Aligned Schema Design

  • Structured core clinical entities using FHIR resource conventions for consistency and interoperability with Epic data exports.
  • Normalized the data model to third normal form (3NF) to enable clean integration and analytical querying.
  • Defined modular tables for clinical indicators, care plans, nutritional guidance, and coaching interactions.

Data Transformation & Modeling Framework

  • Designed a layered data-modeling structure organizing inputs into staging, refined clinical indicators, and longitudinal analytics tables.
  • Developed transformation logic to standardize raw EHR extracts and coaching-app data into consistent representations of HbA1c, BMI, lipids, and blood pressure.
  • Emphasized modularity, traceability, documentation, and structured transformation steps for clarity and reproducibility.
  • Produced analysis-ready longitudinal tables reflecting each patient's clinical timeline across pre- and post-intervention stages.

Integration Architecture

  • Modeled multi-source ingestion from Epic (FHIR-structured extracts) and the coaching application into a unified relational framework.
  • Outlined cloud-compatible ETL logic (Glue-style transformations) before loading into PostgreSQL.
  • Applied HIPAA-aligned design considerations, including identifier separation, minimal PHI exposure, and role-appropriate access patterns.
  • Enabled clinical and coaching data to converge cleanly into a consolidated schema.

Results

  1. Delivered an implementation-ready clinical data architecture: A complete model that integrates EHR and coaching-app data into a unified, structured ecosystem.
  2. Established an interoperable, FHIR-aligned schema: Ensures consistency with Epic and supports future expansion into digital-health ecosystems.
  3. Created longitudinal, analysis-ready data structures: Designed tables that capture pre/post clinical indicators for evaluating program outcomes and patient trajectories.
  4. Aligned schema with real clinical workflows: Modeled interactions among nurse practitioners, health coaches, informaticists, and analysts to ensure operational feasibility.
